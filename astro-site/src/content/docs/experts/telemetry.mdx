Superheld erzeugt strukturierte Telemetriedaten, die Sicherheitsvorfälle dokumentieren. Diese Seite beschreibt die Log-Typen, das kanonische Event-Schema, Datenschutzregeln und Aufbewahrungsfristen.

## Event-Kategorien

Superheld protokolliert sicherheitsrelevante Ereignisse in drei Kategorien:

- **Detection Events** — Erkannte Bedrohungen (Phishing-Versuche, Social-Engineering-Angriffe, schädliche Apps etc.).
- **User Actions** — Sicherheitsrelevante Nutzeraktionen (Freigeben einer blockierten URL, Ignorieren einer Warnung, manuelles Starten eines Scans).
- **Policy Changes** — Änderungen an Sicherheitsrichtlinien (Schwellenwerte, Schutzmodule, Alert-Regeln).

Jedes Event enthält einen ISO-8601-Zeitstempel, eine eindeutige Event-ID und die Geräte-ID des betroffenen Endpunkts.

Das folgende Schema definiert die Struktur eines Detection Events. Dieses Schema gilt auch für die SIEM-Integration (siehe [SIEM-Integration](/experts/siem-integration)).

```json
{
  "event_id": "evt_abc123",
  "timestamp": "2025-01-15T14:32:00Z",
  "device_id": "dev_xyz789",
  "threat_category": "phone_scam",
  "confidence": 0.92,
  "action_taken": "block",
  "policy_id": "pol_default",
  "metadata": {}
}
```

### Feldübersicht

| Feld | Typ | Beschreibung |
|---|---|---|
| `event_id` | string | Eindeutige Kennung des Ereignisses (Präfix `evt_`) |
| `timestamp` | string | ISO-8601-Zeitstempel der Erkennung |
| `device_id` | string | Pseudonymisierte Gerätekennung (Präfix `dev_`) |
| `threat_category` | string | Bedrohungskategorie (siehe unten) |
| `confidence` | float | Konfidenzwert des Erkennungsmodells (0.0–1.0) |
| `action_taken` | string | Automatische Reaktion (`block`, `quarantine`, `warn`) |
| `policy_id` | string | ID der angewandten Schutzrichtlinie (Präfix `pol_`) |
| `metadata` | object | Zusätzliche kontextabhängige Daten |

## Audit Logs

Audit Logs dokumentieren administrative Aktionen und Konfigurationsänderungen innerhalb eines Superheld-Kontos. Relevant für Team- und Enterprise-Accounts.

Protokollierte Aktionen:

- **Konfigurationsänderungen** — Anpassungen an Schutzrichtlinien, Schwellenwerten oder Modellparametern.
- **Zugriffsverwaltung** — Hinzufügen oder Entfernen von Teammitgliedern, Änderungen an Rollen und Berechtigungen.
- **API-Schlüssel-Verwaltung** — Erstellung, Rotation oder Widerruf von API-Schlüsseln.
- **Alert-Konfiguration** — Erstellen, Ändern oder Löschen von Benachrichtigungsregeln.
- **Geräteregistrierung** — Hinzufügen oder Entfernen von Endgeräten.

Jeder Audit-Log-Eintrag enthält: Identität des handelnden Nutzers, Zeitstempel, durchgeführte Aktion, vorheriger und neuer Zustand der geänderten Ressource.

## Redaction und Datenschutz

Superheld wendet automatische Redaktionsregeln an, bevor Telemetriedaten das Gerät verlassen:

- **Personenbezogene Daten (PII)** — E-Mail-Adressen, Namen und Telefonnummern werden durch Platzhalter ersetzt (z. B. `[REDACTED_EMAIL]`).
- **Nachrichteninhalte** — E-Mail-Bodys, Chat-Nachrichten und Dateiinhalte werden nicht in Telemetriedaten aufgenommen.
- **URLs** — Erkannte Phishing-URLs werden als gehashte Werte übertragen. Die vollständige URL verbleibt lokal.
- **IP-Adressen** — Quell-IP-Adressen werden vor der Übertragung auf Netzwerk-Ebene gekürzt (/24 für IPv4, /48 für IPv6).

:::caution
Die Redaktionsregeln greifen ausschließlich bei der Cloud-Übertragung. Lokale Logs auf dem Gerät enthalten die vollständigen, unbereinigten Daten und unterliegen dem lokalen Geräteschutz (AES-256-Verschlüsselung).
:::

## Export an SIEM-Plattformen

Superheld unterstützt den Export von Telemetriedaten an externe SIEM-Systeme (Security Information and Event Management):

- **Webhook-Push** — Superheld sendet Ereignisse in Echtzeit an einen konfigurierbaren HTTPS-Endpunkt. Das Payload-Format entspricht dem kanonischen Event-Schema (siehe oben).
- **Polling-API** — Über den Endpunkt `GET /api/v1/events` können SIEM-Systeme Ereignisse periodisch abrufen. Paginierung und Zeitfilter ermöglichen effiziente Synchronisation.

Details zur SIEM-Integration: [SIEM-Integration](/experts/siem-integration).
